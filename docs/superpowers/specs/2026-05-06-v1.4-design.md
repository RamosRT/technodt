## 2. Feature 1 — 1C Timestamp Marks

### 2.1 Purpose

When an envelope is sealed or verified, mark each document in 1C via OData `InformationRegister_ДополнительныеСведения` so the 1C base reflects the envelope lifecycle without manual entry.

### 2.2 Property Keys (constants in `app/services/odata.py`)

```python
PROP_REGISTERED = uuid.UUID("bda8ba09-4787-11f1-92ca-00155d060d01")  # Зарегистрирован к отправке
PROP_SEALED     = uuid.UUID("d034a826-4787-11f1-92ca-00155d060d01")  # Запечатан
PROP_VERIFIED   = uuid.UUID("daa0fcae-4787-11f1-92ca-00155d060d01")  # Проверен
```

### 2.3 Eligible Document Types

Marks are sent **only** for `Document_СчетФактураВыданный` (УПД / УКД).  
`Document_ПеремещениеТоваров` is silently skipped — no POST/PATCH is sent.

```python
MARK_ELIGIBLE_ENTITIES = frozenset({"Document_СчетФактураВыданный"})
```

### 2.4 When Marks Fire

**On `seal`** — for every eligible document in the envelope, fired in parallel via `asyncio.gather`:
- `PROP_REGISTERED` → value = `doc.added_at` (time document was scanned into draft)
- `PROP_SEALED`     → value = `envelope.sealed_at`

**On `verify/finish`** — for every eligible document that was scanned at verification:
- `PROP_VERIFIED`   → value = `envelope.verified_at`

Marks are **not** sent when documents are added to the draft, avoiding ghost marks for documents later removed.

### 2.5 OData Write Protocol

For each (doc_guid, doc_entity, property_key, value):

1. **POST** to `InformationRegister_ДополнительныеСведения`:
   ```json
   {
     "Объект": "<doc_guid>",
     "Объект_Type": "StandardODATA.<doc_entity>",
     "Свойство_Key": "<property_key>",
     "Значение": "<value as ISO datetime>",
     "Значение_Type": "Edm.DateTime"
   }
   ```
2. If response is **400** with `code="15"` ("запись уже существует") → **PATCH** the existing record:
   ```
   PATCH InformationRegister_ДополнительныеСведения(
     Объект='<doc_guid>',
     Объект_Type='StandardODATA.<doc_entity>',
     Свойство_Key=guid'<property_key>'
   )
   {"Значение": "<value>", "Значение_Type": "Edm.DateTime"}
   ```
3. Any other error → enter retry flow (§2.5).

New method added to `OneCClient`:
```python
async def mark_document(
    self,
    doc_guid: uuid.UUID,
    doc_entity: str,
    property_key: uuid.UUID,
    value: datetime,
) -> None: ...
```

### 2.6 Background Execution & Retry

Marks run fully in the background — the API response to `seal` / `verify/finish` is **not blocked**.

```python
asyncio.create_task(
    _mark_documents_with_retry(one_c, docs, props, settings)
)
```

Inside `_mark_documents_with_retry`:
1. First attempt — `asyncio.gather` all marks for the event.
2. On any exception → `await asyncio.sleep(5)` → one retry.
3. If retry also fails → `logging.warning(...)` + write `audit_log` event `onec_mark_failed` with payload `{doc_guid, property_key, error_message}`.

### 2.7 Feature Flag

`.env` key: `ENABLE_1C_TIMESTAMPS=true` (default `true`).  
Added to `Settings` in `app/config.py`.

Admin can also toggle it via admin UI (stored as a DB setting row, takes priority over env if present). Toggle is visible in the Settings drawer for admin users (§3.4).

## 3. Feature 2 — Print Subsystem v2

### 3.1 Printer Storage — Move to DB

Replace `PRINTERS_JSON` env var with a `printers` DB table managed via admin UI.

#### Schema

```python
class Printer(Base):
    __tablename__ = "printers"

    id:         Mapped[str]       # slug, e.g. "tlp300-sklad" — PK
    name:       Mapped[str]       # human-readable label
    kind:       Mapped[str]       # "zpl" | "a4"
    is_active:  Mapped[bool]      # visible to operators; default True
    host:       Mapped[str|None]  # IP for ZPL TCP socket
    port:       Mapped[int|None]  # port for ZPL (usually 9100)
    dpi:        Mapped[int|None]  # ZPL DPI: 200 or 300
    share_name: Mapped[str|None]  # Windows share name for A4 (\\10.60.6.11\<share_name>)
    created_at: Mapped[datetime]
```

Alembic migration: create table, seed from `PRINTERS_JSON` if table is empty on first run. After migration `PRINTERS_JSON` is ignored.

### 3.2 Operator Preferences

Add three fields to `Operator` to persist user workspace defaults:

```python
assigned_zpl_printer_id:   Mapped[str | None]        # existing
assigned_a4_printer_id:    Mapped[str | None]        # new — ref to printers.id
default_branch_id:         Mapped[uuid.UUID | None]  # new — preferred origin branch
default_signer_sender_id:  Mapped[uuid.UUID | None]  # new — preferred signatory
```

Alembic migration: three `ADD COLUMN` statements (`VARCHAR(100)`, `UUID`, `UUID`).

### 3.3 A4 Printing via win32print

PDF download (existing `GET /api/envelopes/{id}/print/inventory`) is unchanged.

New backend path — send PDF directly to print server:

Print server host configured via `.env` key `PRINT_SERVER_HOST` (default `10.60.6.11`), added to `Settings`.

```python
def _win32print_send(share_name: str, data: bytes, *, host: str) -> None:
    import win32print
    path = rf"\\{host}\{share_name}"
    handle = win32print.OpenPrinter(path)
    try:
        win32print.StartDocPrinter(handle, 1, ("konvert-inventory", None, "RAW"))
        win32print.StartPagePrinter(handle)
        win32print.WritePrinter(handle, data)
        win32print.EndPagePrinter(handle)
    finally:
        win32print.EndDocPrinter(handle)
        win32print.ClosePrinter(handle)

async def send_inventory_to_a4_printer(
    session: AsyncSession,
    envelope_id: uuid.UUID,
    printer: Printer,
) -> None:
    pdf_bytes = await render_inventory_pdf(session, envelope_id)
    await asyncio.to_thread(_win32print_send, printer.share_name, pdf_bytes)
```

`pywin32` added to dependencies (already available on the Windows host).

### 3.4 API Changes

| Method | Path | Notes |
|---|---|---|
| `GET` | `/api/printers` | Active printers only (for operators) — replaces old env-based endpoint |
| `GET` | `/api/admin/printers` | All printers including inactive (admin only) |
| `POST` | `/api/admin/printers` | Create printer |
| `PATCH` | `/api/admin/printers/{id}` | Edit / toggle `is_active` |
| `DELETE` | `/api/admin/printers/{id}` | Delete |
| `POST` | `/api/envelopes/{id}/print/inventory/send?printer_id=…` | Send inventory PDF to A4 printer (was 501, now implemented) |
| `PATCH` | `/api/operators/me/settings` | Save operator workspace preferences (printers, branch, signer) |

### 3.5 Settings Drawer (Web UI)

Replace the bell icon in the navbar with a gear icon `⚙️`. Clicking it opens a right-side drawer (does not navigate away from current page).

Drawer sections:

1. **Рабочее место** — branch selector, signatory selector (dropdowns from existing API)
2. **Принтеры** — ZPL printer dropdown (active ZPL printers), A4 printer dropdown (active A4 printers); each shows a badge with kind and address
3. **Система** *(admin only)* — toggle for "Отметки времени в 1С" (`ENABLE_1C_TIMESTAMPS`)

Save button calls a single `PATCH /api/operators/me/settings` endpoint with body:
```json
{
  "zpl_printer_id": "...",
  "a4_printer_id": "...",
  "default_branch_id": "...",
  "default_signer_sender_id": "..."
}
```
All fields optional; null clears the preference. Response returns the full updated operator profile. Selections persist server-side and pre-fill on next login (seal modal, settings drawer).

### 3.6 Admin Printer Management UI

New admin section "Принтеры" in the existing admin sidebar.

- Table: name, slug ID, kind badge (ZPL/A4), address/share, visibility toggle, edit/delete actions
- "Добавить принтер" button opens a modal:
  - Fields: name, slug, kind (select), IP+port+DPI (ZPL) or share name (A4)
- Visibility toggle calls `PATCH /api/admin/printers/{id}` with `{"is_active": bool}`

### 3.7 TSD — A4 Print Button

Add an **«Опись А4»** button next to the existing ZPL label button in the envelope detail screen.

- Calls `POST /api/envelopes/{id}/print/inventory/send?printer_id={operator.assigned_a4_printer_id}`
- If `assigned_a4_printer_id` is null → show a printer picker bottom sheet (list of active A4 printers)
- On success → show a brief snackbar "Опись отправлена на печать"

## 4. Feature 3 — TSD Compatibility

### 4.1 Mindeo M40 — Scanner Integration

**Root cause:** M40 uses `com.hyatta.hscan` (OEM Hyatta, system service) which fires a standard Android Broadcast — not a DataWedge Intent API used by Zebra/Honeywell.

**Confirmed via ADB logcat + HScan.apk analysis:**

| Parameter | Value |
|---|---|
| Broadcast action | `com.android.scanner.broadcast` |
| Barcode data extra key | `"scandata"` (String) |
| Barcode type extra key | `"barcode_type"` (**Byte** — not String/Integer) |
| QR symbology code | `62` |
| Decode result (success) | `0` |

**New adapter:**

```kotlin
class MindeoAdapter(private val context: Context) : ScannerAdapter {
    private var receiver: BroadcastReceiver? = null

    override fun start(onScan: (String) -> Unit) {
        receiver = object : BroadcastReceiver() {
            override fun onReceive(ctx: Context, intent: Intent) {
                val barcode = intent.getStringExtra("scandata") ?: return
                if (barcode.isNotBlank()) onScan(barcode)
            }
        }
        context.registerReceiver(
            receiver,
            IntentFilter("com.android.scanner.broadcast"),
            Context.RECEIVER_EXPORTED
        )
    }

    override fun stop() {
        receiver?.let { context.unregisterReceiver(it) }
        receiver = null
    }
}
```

**Adapter selection** in `ScannerAdapterFactory`:

```kotlin
fun create(context: Context): ScannerAdapter = when {
    Build.MANUFACTURER.lowercase() in listOf("mindeo", "hyatta") -> MindeoAdapter(context)
    else -> DataWedgeAdapter(context)  // existing
}
```

Manual override available in the service settings screen (persisted in DataStore).

**QR auth fix:** The login screen uses the same `ScannerAdapter` interface — no additional changes needed; `MindeoAdapter` will handle QR scan on the login screen automatically.

### 4.2 Urovo DT40 — Layout Fixes

**Root cause:** Some Compose screens do not handle narrow `screenWidthDp` values correctly — elements overlap on DT40's screen.

**Fix:** Apply `WindowSizeClass` breakpoints across affected screens:

```kotlin
val windowSizeClass = calculateWindowSizeClass(activity)
val isCompact = windowSizeClass.widthSizeClass == WindowWidthSizeClass.Compact
```

Affected areas to audit and fix:
- Document list cards in envelope detail
- Action button rows (seal / verify buttons)
- Header info block (envelope number + barcode)

Use `isCompact` to switch between single-column and multi-column layouts where needed.
