# ТехноКонверт v1.1 — Design Spec

**Date:** 2026-04-30  
**Status:** Approved  
**Scope:** Builds on top of MVP (FastAPI + HTMX + PostgreSQL, already deployed via nssm/uvicorn).

---

## 1. Goals

Three feature groups shipped in v1.1:

1. **Envelope list + Document list** — read-only reporting views with filters, accessible from the new navbar.
2. **Audit log in UI** — timeline embedded in the envelope card (all operators) + full admin screen.
3. **Operator roles + Unseal** — named operator registry with `is_admin` flag; admin can revert a sealed envelope to draft, edit its composition, and reseal.

---

## 2. Navigation — Persistent Top Navbar

The current `mode-selector` card grid on `index.html` is replaced by a persistent top navbar rendered in `base.html`.

**Navbar layout (left → right):**
- Company logo (`icon.png`, 32×32, rounded) + text **ТехноКонверт**
- Nav items: **Регистрация** (`package-plus`), **Верификация** (`scan-line`), **Конверты** (`archive`), **Документы** (`file-text`)
- Right side: operator chip (`circle-user` + name), Admin badge (red gradient, visible only when `is_admin=true`), Settings gear (`settings-2` → operator/admin management), Logout (`log-out`)

**Active item** highlighted with a bottom border in light blue (`#6ea9dc`) and a subtle white overlay.

Icons: [Lucide Icons](https://lucide.dev/) via CDN (`https://unpkg.com/lucide@latest/dist/umd/lucide.js`), already added to `base.html`. Use `<i data-lucide="...">` + `lucide.createIcons()` call.

**HTMX routing:** each nav item fires an HTMX request that swaps `#mode-area`. No full page reload.

## 3. Operator Registry & Auth

### 3.1 New table: `operators`

```sql
CREATE TABLE operators (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username   VARCHAR(200) NOT NULL UNIQUE,
    is_admin   BOOLEAN NOT NULL DEFAULT FALSE,
    is_active  BOOLEAN NOT NULL DEFAULT TRUE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

Alembic migration required.

### 3.2 Auth logic changes

`require_operator()` behavior:
- If `operators` table is **empty** → accept any non-empty name (backwards-compatible bootstrap mode).
- If table is **non-empty** → accept only names where `is_active=true` exists in the table. Return 401 with a friendly message otherwise.

`require_admin()` behavior:
- Replace the static `X-Admin-Token` header check with a lookup: cookie `operator_name` must match a row with `is_admin=true` and `is_active=true`.
- The old `ADMIN_TOKEN` config key is removed.

### 3.3 Bootstrap

New env var: `BOOTSTRAP_ADMIN=<username>`. On every login, if the incoming name matches `BOOTSTRAP_ADMIN`, ensure a row exists in `operators` with `is_admin=true` (upsert). This lets the first admin log in without touching the DB directly.

### 3.4 Operator management UI

Accessible via the settings gear icon → loads `partials/operators.html` into `#mode-area`.

- **Admin-only screen.** Non-admins see a 403 partial.
- Table of all operators: name, is_admin toggle, is_active toggle, created date.
- Add operator form (name field + is_admin checkbox).
- Deactivate button (soft delete — sets `is_active=false`).
- An admin cannot deactivate themselves.

## 4. Envelope List

### 4.1 Route

`GET /ui/envelopes` → renders `partials/envelopes_list.html`

### 4.2 Filters

| Filter | Input type | Notes |
|---|---|---|
| Status | `<select>` | All / Draft / Sealed / Verified / Discrepancy |
| Date from | `<input type="date">` | Filters `created_at` |
| Date to | `<input type="date">` | Inclusive |
| Branch | `<select>` | All active branches |
| Search | `<input type="text">` | Matches envelope number or barcode prefix |

Applied on form submit (HTMX `hx-get`). Reset button clears all fields and reloads.

### 4.3 Table columns

| Column | Notes |
|---|---|
| № конверта | Monospace, links to envelope card |
| Статус | Pill with icon (circle-check / triangle-alert / lock / pencil) |
| Документов | Count; if `verified_with_discrepancy`, show `−N` in red |
| Оператор | `created_by` |
| Создан | `created_at` formatted `DD.MM.YYYY HH:MM` |
| Филиал-отправитель | `origin_branch.name` or `—` |
| (action) | «Открыть →» link |

Rows with `verified_with_discrepancy` get a light red background (`#fff9f9`).

### 4.4 Pagination

Server-side. Page size: 25. Query params: `?page=1`. HTMX swaps just the table+pagination partial on page change.

## 5. Document List

### 5.1 Route

`GET /ui/documents` → renders `partials/documents_list.html`

### 5.2 Summary chips (top of page)

Four read-only chips computed from the current filter result:

- **Всего за период** — total document count
- **Верифицированы** — `scanned_at_verification IS NOT NULL`
- **В пути** — envelope status is `sealed`, document not yet scanned at verification
- **Недостача** — envelope status is `verified_with_discrepancy` AND `scanned_at_verification IS NULL`

### 5.3 Filters

| Filter | Input type |
|---|---|
| Date from / to | `<input type="date">` (filters `added_at`) |
| Doc type | `<select>` (УПД / УКД / Перемещение / All) |
| Status | `<select>` (Верифицирован / В пути / Недостача / All) |
| Branch | `<select>` |
| Search | Text — matches doc number or barcode |

Plus an **Экспорт** button that calls `GET /api/documents/export?<same filters>` → returns CSV.

### 5.4 Table columns

| Column | Notes |
|---|---|
| Тип | Badge (УПД / УКД / ПТ) |
| № документа | Bold |
| Дата документа | `doc_date` |
| Конверт | Envelope number, links to envelope card |
| Добавлен | `added_at` + `created_by` of the envelope (two-line cell) |
| Запечатан | `sealed_at` of the envelope (two-line: date + operator) |
| Верифицирован | `scanned_at_verification` + `verified_by` of envelope; `— ожидает` or `— не отсканирован` when null |
| Статус | Pill: Верифицирован / В пути / Недостача |
| (action) | Eye icon, opens envelope card |

Rows with "Недостача" get light red background. Rows "Верифицирован" get light green background.

Pagination: same pattern as envelope list, page size 50.

## 6. Unseal Operation

**Who:** admin only. Non-admins do not see the panel.

**When visible:** envelope card opened from the list while status is `sealed` or `verified` / `verified_with_discrepancy`. (Draft envelopes are already editable.)

### 6.1 Flow

1. Admin sees a yellow dashed panel: "Конверт запечатан. Распечатайте его, чтобы отредактировать состав."
2. Clicks **Распечатать** → modal appears with:
   - Warning text explaining the action and that it is logged.
   - Required text field: **Причина распечатки**.
   - Buttons: Отмена / Подтвердить распечатку (red).
3. On confirm → `POST /api/envelopes/{id}/unseal` with `{ "reason": "..." }`.
4. Envelope status → `draft`. Audit event `unseal` written with actor + reason in payload.
5. Envelope card reloads. Status pill shows `draft`. Document table now shows **delete (×) buttons** on each row (same as during original registration). The full seal form (signers, branches) reappears at the bottom.
6. Admin edits composition (add/remove documents), then seals again via the existing seal flow. A new `seal` event is written to audit log.

### 6.2 New API endpoint

```
POST /api/envelopes/{id}/unseal
Body: { "reason": "string" }
Auth: require_admin
Returns: EnvelopeOut
```

Service: sets `status=draft`, clears `sealed_at`, writes audit event `unseal`.

## 7. Audit Log

### 7.1 Embedded in envelope card

- Collapsible section at the bottom of every envelope card. Visible to all operators.
- Trigger: clicking the "История событий" header toggles the timeline.
- Shows all `audit_log` rows for this `envelope_id`, ordered newest-first.
- Each event: colored dot + event title (human-readable) + actor + timestamp + optional detail line from `payload`.

Color coding of dots:
| Event | Dot color |
|---|---|
| `create` | Blue (`#2874b9`) |
| `add_doc`, `remove_doc` | Light blue (`#6ea9dc`) |
| `seal` | Orange (`#f57f17`) |
| `unseal` | Red (`#cc0c00`) |
| `verify_start`, `verify_scan`, `verify_finish` | Green (`#2e7d32`) |

### 7.2 Admin audit screen

- Accessible via settings gear → "Аудит-лог" menu item.
- Full table of all audit events across all envelopes.
- Filters: date range (`<input type="date">`), event type, actor (operator name), envelope number/barcode search.
- Columns: timestamp, event, конверт (link), actor, payload summary.
- Pagination, page size 50. No export in v1.1 (deferred to v1.2).
